# LeetCode Tracker - Frontend Environment Variables

This file contains the environment variables needed to run the frontend application.

## Setup Instructions

1. Copy this file to `.env.local`:

   ```bash
   cp .env.example .env.local
   ```

2. Get your AWS resources information from CDK outputs:

   ```bash
   cd ../../infra
   npx cdk deploy --all --outputs-file outputs.json
   ```

3. Fill in the values:
   - **VITE_API_URL**: Your API Gateway endpoint (from BackendStack output)
   - **VITE_COGNITO_USER_POOL_ID**: Your Cognito User Pool ID (from AuthStack)
   - **VITE_COGNITO_CLIENT_ID**: Your Cognito User Pool Client ID (from AuthStack)
   - **VITE_COGNITO_REGION**: AWS region (e.g., us-east-1)

## Getting Values from AWS Console

### API Gateway URL:

1. Go to AWS Console → API Gateway
2. Select "LeetCode Tracker API"
3. Go to "Stages" → select your stage (dev/prod)
4. Copy the "Invoke URL"

### Cognito User Pool ID & Client ID:

1. Go to AWS Console → Cognito
2. Select your User Pool ("leetcode-tracker-users-dev")
3. User Pool ID is shown at the top
4. Go to "App integration" tab
5. Find your app client and copy the "Client ID"

## Example Values

```env
VITE_API_URL=https://abc123xyz.execute-api.us-east-1.amazonaws.com/dev
VITE_COGNITO_USER_POOL_ID=us-east-1_ABC123XYZ
VITE_COGNITO_CLIENT_ID=1a2b3c4d5e6f7g8h9i0j1k2l3m
VITE_COGNITO_REGION=us-east-1
```

## Important Notes

- Never commit `.env.local` to version control
- The `.env.example` file should NOT contain real values
- All variables must start with `VITE_` to be exposed to the browser
